# AGENTS.md — src/tdt/ui (SP4 UI Desktop)

## Purpose
Interface gráfica PySide6 para configurar, executar e revisar o pipeline SP1. Input `.xlsx` → revisão interativa → `TDT.xlsx`.

## Ownership
- `app.py`: `MainWindow(QMainWindow)` — Sidebar retrátil + QStackedWidget (0 Entrada, 1 Revisão, 2 Config, 3 Análise, 4 Geração); navegação por chave via `_navegar`; undo global Ctrl+Z; gating de etapas pós-execução.
- `tela_inicial.py`: `TelaInicial(QWidget)` — seleção de arquivos (input, template, lista, output), botão executar, barra de progresso. `definir_marcacao(lista, marcado)`/`inverter_marcacao(lista)` (módulo-level, puras) marcam/desmarcam/invertem checkboxes de `lista_sheets` em grupo — agem nas linhas selecionadas, sem seleção agem em todas; expostas via menu de contexto, tecla Espaço e hitbox de linha inteira (clique fora do quadradinho também alterna, exceto com shift/ctrl).
- `tela_revisao.py`: `TelaRevisao(QWidget)` — tabela de revisão com `ProxyRevisao` + `DelegateSinal` + `ModeloSinais`; filtro de coluna unificado (popup com campo "contém" + valores estilo Excel); segmented de status (Todos/Pendentes/Decididos); fluxo de teclado aprovar-e-avançar (Enter/1-5). Botão "Reparear sheet" (SP-OBS-17JUL P3, `_reparear_sheet`) roda `AppState.aplicar_reparear` na sheet inteira ou só na seleção (≥2 linhas pergunta), whitelist de posição via `tdt.pipeline._whitelist_posicao(lp, config)` (import lazy — puxar no topo do módulo quebra o boot leve da UI, ver `worker.py`). Botão "Separar par de posição" (SP-OBS-17JUL P3b, `_separar_par_posicao`) chama `AppState.separar_par_posicao` numa seleção de 1 linha, com confirmação mostrando os endereços reais.
- `tela_config.py`: `TelaConfig(QWidget)` — knobs de configuração do pipeline (Config paths).
- `tela_analise.py`: `TelaAnalise(QWidget)` — análise de qualidade do matching pós-pipeline.
- `sidebar.py`: `Sidebar(QWidget)` — navegação retrátil (48/200px, persiste em QSettings("tdt","ui")); estados por item (disponivel/bloqueada/completa), badge de pendentes, contexto no rodapé.
- `tela_geracao.py`: `TelaGeracao(QWidget)` — etapa 3: resumo, avisos acionáveis (pendentes, endereços duplicados), confirmações e geração do `TDT_<SUB>_<YYYYMMDD>.xlsx` + `Auditoria_<SUB>_<YYYYMMDD>.xlsx` (nomes via `tdt.nomes_saida.nome_saida`, sufixo `_vN` em colisão; movida da TelaRevisao). Importa `tdt.pipeline` de forma lazy dentro de `_gerar` (boot da UI não puxa sklearn — ver `worker.py`). `filtrar_por_modulos(registros, marcados)` (módulo-level, pura) — `marcados` é um set de nomes de módulo (`None` = "(sem módulo)"). `lista_modulos` (QListWidget checkable, todos marcados por default) + `_modulos_marcados()` restringem geração/relatório/contadores/avisos ao MESMO subconjunto filtrado; sem módulo marcado desabilita `btn_gerar`.
- `estado.py`: `AppState` — estado mutável compartilhado entre telas (Config, ResultadoPipeline, ListaPadraoADMS, motivo revisão). `_editar_nested` + 9 setters (`definir_tipo/fase/nivel_tensao/barra/tipo_equip/modulo/escala/equipamento/descricao_bruta`) editam campos aninhados de `SignalRecord` via `dataclasses.replace`. `aprovar_ids(ids)` (SP-CVA Task 10) aprova a seleção inteira com um único snapshot (um Ctrl+Z desfaz o lote). `formar_par_posicao(id_a, id_b, sigla)`/`trocar_sigla_par(id_, nova_sigla)` (SP-CVA Task 15) — caminho do operador pra corrigir DJA1↔DJF1: funde dois registros em `MultiCoord` (valida mesmo módulo/equipamento, 1 endereço cada) ou troca a sigla de um par já fundido preservando os endereços. `definir_equipamento(indice, nome|None)` (SP-UX-PERF Task 4) corrige o ID do equipamento (`eletrico.nome_equipamento`); `definir_descricao_bruta(indice, texto)` só troca `descricoes.bruta` — normalizada/tokens NÃO reprocessam. `aplicar_reparear(ids, whitelist_posicao, config)` (SP-OBS-17JUL P3) reparea em lote os elegíveis dentre `ids` via `ui/reparear.py` (núcleo puro, reusa `dc_pairer.parear`+`normalizador_estrutural.fundir_pares_posicao`) — 1 snapshot p/ o lote inteiro, ordem preservada, comando absorvido pela fusão sai da lista. `separar_par_posicao(id_)` (SP-OBS-17JUL P3b) — inverso de `formar_par_posicao`, mas **RECONSTRÓI** (não desfaz): quebra um par de posição já fundido (Input MultiCoord 2 endereços) em 2 Inputs SingleBit de 1 endereço, sigla limpa, `status="revisao"`, `justificativa="posicao_ambigua"` — o dado do registro absorvido na fusão original não existe mais (limitação documentada, spec §9.6; **FOLLOW-UP**: preservar esse dado é fora de escopo do plano atual).
- `worker.py`: `PipelineWorker(QThread)` — isola o pipeline em QThread; cancelamento cooperativo via flag. `encoder_factory`/`executar_fn` default `None` e resolvidos lazy dentro de `run()` (`criar_encoder`/`pipeline.executar`) — o import pesado (sentence_transformers/sklearn) só acontece ao rodar análise, nunca no boot da UI. Testes injetam os factories explicitamente.
- `reparear.py` (SP-OBS-17JUL P3): núcleo puro do repareamento em lote — `elegivel(rec)`/`reparear(elegiveis, whitelist_posicao, config) -> ResultadoReparear`, reusa `dc_pairer.parear`+`fundir_pares_posicao`, sem widgets, sem estado próprio (input/output puros).
- `modelo_tabela.py`: `ModeloSinais(QAbstractTableModel)` — tabela de revisão (colunas: Módulo, End., Sigla, Decisão, etc.). `flags()`/`setData()` habilitam edição em 10 colunas (Sinal + 9 campos de domínio, incl. Equipamento e Descr. bruta — SP-UX-PERF Task 4), dispatchando pros setters do `AppState`. Colunas "Endereço Input"/"Endereço Output" (SP-OBS-17JUL P1, renomeada de "Endereço"; `enderecos_exibicao(rec)` decide qual célula mostra o quê por direção — Output puro tem endereço PRÓPRIO de escrita, mostrado em "Endereço Output" com "Endereço Input"="—"; edição espelha a mesma regra em `setData`/`_valor_edicao`, sem vazar o sentinela "—" pro editor) editáveis (SP-CVA Task 12: parser `900`/`900;901`, valida int 0..65535, recusa sem mutação se inválido). Coluna "Severidade" (SP-OBS-17JUL 2D) não-editável, metadado puro da LP. Motivo (`_texto`) tem fallback: `motivo_por_id().get(rec.id)` (derivado do `resultado.revisao` congelado da última execução) OU `rec.justificativa` (campo mutável, usado por edições pós-pipeline como `separar_par_posicao` e pelos ambíguos do `reparear`) — sem esse fallback, motivos atribuídos DEPOIS do pipeline rodar não apareciam na UI. Colunas "Pareado" (`Sim`/`Órfão`/`—`) e "Sheet origem" (SP-CVA Task 13, `sheet_origem(rec)`). `valor_texto(row, col)` (SP-UX-PERF Task 2) — acessor de exibição sem `QModelIndex`, espelha o branch `Qt.DisplayRole` de `data()`; caminho quente do filtro. `aplicar_valor_em_lote(ids, coluna, valor)` (SP-CVA Task 11) propaga edição a várias linhas com um único snapshot; desde SP-UX-PERF Task 5 suprime snapshot/`dataChanged` por linha durante o loop e emite **um** `dataChanged` agregado no fim. `contagem_por_sheet()` (SP-CVA Task 14) devolve `(pendentes, total)` por sheet pro rótulo das abas. `remover_linhas(indices)` (SP-UX-PERF Task 5): lote > `LIMIAR_RESET_REMOCAO` (100) usa `beginResetModel`/`endResetModel` único (O(n)); lote ≤ 100 usa `beginRemoveRows`/`endRemoveRows` por range contíguo (`_ranges_contiguos`, módulo-level), em ordem reversa.
- `modelo_analise.py`: `ModeloAnalise(QAbstractTableModel)` — tabela de análise de qualidade.
- `proxy_revisao.py`: `ProxyRevisao(QSortFilterProxyModel)` — filtra por coluna com 2 mecanismos combinados em AND (texto "contém" + valores estilo Excel); `set_status_visivel(None|"revisao"|"decidido")` controla visibilidade por status. `filterAcceptsRow` usa `modelo.valor_texto(row, col)` (caminho quente, sem `QModelIndex`) e roda os filtros baratos (sheet/status/valores/contém) antes de `super().filterAcceptsRow` (busca global de texto, mais caro — SP-UX-PERF Task 2).
- `delegate_sinal.py`: `DelegateSinal(QStyledItemDelegate)` — editor combo da coluna Sigla (candidatos + busca ADMS). `DelegateCombo` — combo de opções fixas p/ colunas de domínio fechado (Tipo/Fase/Nível Tensão/Barra/Tipo Equip.). `DelegateModulo` — combo editável p/ Módulo, sugere nomes já presentes nos registros. `DelegateEquipamento` (SP-UX-PERF Task 4) — combo editável p/ Equipamento (ID), sugere IDs já presentes em registros do MESMO módulo (via `proxy.mapToSource`); texto livre aceito, vazio limpa. "Descr. bruta" usa o `QLineEdit` default do Qt (sem delegate próprio). Os 4 implementam `setEditorData` (pré-seleciona o valor atual da célula ao abrir; `_preselecionar` mapeia o sentinela de exibição "—" pra opção vazia "").
- `busca_adms.py`: busca textual na lista padrão ADMS.
- `exportar_analise.py`: exporta relatório de análise para `.xlsx`.
- `config_io.py`: carrega/salva configuração do pipeline (`config.toml`+paths). `salvar_config` (SP-OBS-17JUL P4) grava SÓ os campos que divergem de `Config()` default — `[config]`/`[pesos_regras]` ficam ausentes do arquivo quando tudo é default. `carregar_config` migra (descarta) o trio de pesos stale `peso_tfidf=0.34/peso_vetorial=0.33/peso_fuzzy=0.33` (snapshot pré-calibração BM25 03jul) só quando os 3 batem juntos — 1 ou 2 batendo é override legítimo do usuário, preservado. `tela_config.py` mostra label de override (`_atualizar_overrides`, quais knobs ≠ default) e bloqueia `aplicar()` se a soma dos 3 pesos do ensemble ≠ 1.0 (guard de soma); escolha de path (`_escolher`) persiste via `salvar_config` direto, sem passar pelo guard de pesos (não deve ficar bloqueada por um ensemble mid-edição inválido).
- `tema.qss`: folha de estilos.
- `app.py` + `ui_main.py`: entry-point da UI.

## Local Contracts
- AppState é mutável e compartilhado entre telas via referência. `motivo_por_id()` devolve motivo de revisão por id do SignalRecord.
- Campos editáveis na tabela de revisão: Sigla, Tipo, Fase, Nível Tensão, Barra, Tipo Equip., Módulo, Escala, Equipamento (ID), Descr. bruta. Editar qualquer um dos campos de domínio NÃO promove `status` pra `"decidido"` (só editar Sigla promove); snapshot de undo é criado DENTRO de `definir_sigla`/`_editar_nested` (lacuna fechada em SP-UI-1); `definir_sigla(..., snapshot=False)` existe para lotes (aprovação automática). Editar Tipo Equip. grava `equipamento_inferido=False`; editar Tipo grava `categoria_confiavel=True`. Domínio fechado de Tipo exclui `DiscreteAnalog` (placeholder de incerteza do dual-pass, não é estado-alvo de edição manual). Ver spec `docs/superpowers/specs/2026-07-01-sp-campos-editaveis-revisao-design.md`. Equipamento (ID) e Descr. bruta (SP-UX-PERF Task 4) são os únicos campos editáveis fora do domínio "classificação": Equipamento corrige o ID (`81-1`→`52-11`), Descr. bruta só afeta o fallback de Signal Alias no export — nenhum dos dois reprocessa matching/normalização. Campos derivados continuam travados: Confiança, Status, Motivo, Descr. lista padrão, Signal Name, Device Mapping, Signal Type, Scores, Justificativa, Pareado, Sheet origem. Signal Name/Device Mapping/Signal Type (spec §D1) são calculadas ao vivo em `_texto` via `dm_registro`/`disjuntor_por_modulo` (`tdt.engine_tdt`, Tasks 1-2) — não há campo persistido no `SignalRecord`, então recalculam sozinhas a cada `data()`.
- PipelineWorker usa `pyqtSignal` para progresso (int), log (str), resultado (ResultadoPipeline) e erro (str). Cancela via `_cancelado` flag com cooperativa.
- `ProxyRevisao` herda `QSortFilterProxyModel`; filtra por coluna (texto "contém" + valores estilo Excel, AND) com marcador " ▼*" no header; `set_status_visivel(None|"revisao"|"decidido")` substitui o antigo esconder_decididos; `filtros_ativos()`/`limpar_filtros()` alimentam o chip "limpar todos".
- Navegação: Sidebar (chaves entrada/revisao/config/analise/geracao) + QStackedWidget de 5 telas; Revisão/Geração/Análise bloqueadas até o primeiro `executou`. Preferências de UI (sidebar, colunas, splitter) em QSettings("tdt","ui").
